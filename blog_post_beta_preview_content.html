Facilitair is an AI orchestration platform that sits between your application and the AI providers. But unlike simple routing proxies, we provide:

Intelligent Model Routing

Our V13 routing model analyzes your task and selects the optimal model based on quality requirements, cost constraints, and latency needs. Not a lookup table - an actual ML model trained on 10,000+ task executions.

350+ Models, One API

Access OpenAI, Anthropic, Google, Mistral, DeepSeek, Grok, Llama, Qwen, and dozens more through a single endpoint. Switch providers without changing code.

200+ Integrated Tools

Built-in Fine-Tuning

Train custom models on your data with GPU provisioning handled automatically. Upload datasets, configure training, monitor progress - all through the same interface.

BYOK (Bring Your Own Keys)

Cost Controls & Budgets

Four Ways to Use Facilitair

Everything in Facilitair is accessible through multiple interfaces. Choose what fits your workflow:

1. Web Dashboard

The fastest way to explore. Visual model comparison, real-time routing visualization, training job monitoring, and interactive task execution.

Best for: Exploration, monitoring, one-off tasks, team collaboration

URL: beta.facilitair.ai/dashboard

2. CLI (corch)

Full-featured command-line interface for developers who live in the terminal. Supports streaming, file uploads, training jobs, and shell completion.

# Install pip install facilitair # Run a task with intelligent routing corch run "Generate a React component for user authentication" --strategy auto # Force a specific model corch run "Explain quantum entanglement" --model anthropic/claude-opus-4.5 # List available models with rankings corch models --category coding --limit 10 # Start a training job corch training create --base-model meta-llama/llama-3.1-8b --dataset my_data.jsonl # Stream logs from running training corch training logs job_abc123 --follow # Execute MCP tools corch services execute slack --action post_message --channel general --message "Deployment complete"

Best for: Automation, scripting, CI/CD integration, power users

3. REST API

OpenAI-compatible API that works as a drop-in replacement. Point your existing code at Facilitair and gain access to all providers.

POST https://api.facilitair.ai/v1/tasks { "prompt": "Write a Python function to detect anomalies in time series data", "strategy": "auto", // Let routing model decide "quality_weight": 0.8, // Prioritize quality over cost "budget_cents": 50, // Max spend on this task "tools": ["github", "postgresql"] // Enable specific MCP tools } // Response includes model used, cost, tokens, and result { "task_id": "task_xyz789", "model_used": "anthropic/claude-sonnet-4.5", "cost_cents": 12, "result": "def detect_anomalies(data, window_size=20, threshold=3.0):..." }

Best for: Application integration, backend services, existing OpenAI SDK users

4. Python SDK

Native Python library with async support, streaming, and type hints. Feels like using the official provider SDKs but with superpowers.

from facilitair import Facilitair client = Facilitair(api_key="your_key") # Simple completion response = client.complete( prompt="Summarize the key points of this document", strategy="auto", budget_cents=25 ) print(response.result) print(f"Model: {response.model_used}, Cost: ${response.cost_cents/100:.4f}") # Streaming async for chunk in client.stream("Write a story about a robot learning to paint"): print(chunk, end="") # Execute MCP tools result = client.tools.execute( service="jira", action="create_issue", params={"project": "ENG", "summary": "Bug: Login timeout", "type": "Bug"} ) # Fine-tuning job = client.training.create( base_model="meta-llama/llama-3.1-8b", dataset_path="training_data.jsonl", config={"epochs": 3, "learning_rate": 2e-5} ) print(f"Training job started: {job.id}")

Best for: Python applications, data science workflows, complex orchestration

Intelligent Routing: The V13 Model

At the heart of Facilitair is our routing model - a classifier trained on 10,000+ task executions that predicts the optimal model for each request. It considers:

  • Task Complexity: Simple Q&A vs multi-step reasoning vs code generation
  • Quality Requirements: Your specified quality_weight (0.0-1.0)
  • Cost Constraints: Your budget limits and cost sensitivity
  • Latency Needs: Real-time vs batch processing
  • Safety Triggers: Automatic consensus for destructive operations

The routing model achieves 54.1% improvement in task completion quality compared to single-model approaches, while often reducing cost by routing simple tasks to efficient models.

Routing Strategies

  • auto: Let the V13 model decide (recommended)
  • direct: Fastest, single model, cost-optimized
  • progressive: Try cheap models first, escalate if needed
  • collaborative: Multiple models for planning, implementation, testing
  • consensus: Require agreement from 3+ models (for safety-critical tasks)

Fine-Tuning: Train Your Own Models

Facilitair isn't just for inference - we provide full model training capabilities:

# Create a training job corch training create \ --base-model meta-llama/llama-3.1-8b \ --dataset ./customer_support_data.jsonl \ --epochs 3 \ --learning-rate 2e-5 \ --provider facilitair # We provision GPU automatically # Monitor progress corch training logs job_abc123 --follow # Check status corch training get job_abc123 # List all jobs corch training list --status running

Training jobs run on provisioned GPU infrastructure (currently H200 GPUs via DigitalOcean). You can also bring your own compute through Tinker Mode for customers with existing GPU resources.
