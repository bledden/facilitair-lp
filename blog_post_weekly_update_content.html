What It Is



Facilitair is an AI orchestration platform that automatically selects the best execution strategy based on task complexity:



    

  • DIRECT: Single model, single call (fast, cheap) - for simple queries
    • 

  • BATCH: Parallel processing of independent subtasks - for list operations
    • 

  • ORCHESTRATE: Sequential multi-step workflow - for complex code generation
    • 

  • COLLABORATE: Multi-model collaboration with synthesis - for open-ended research
    • 

  • CONSENSUS: Multiple models vote on best answer - for critical decisions
    •

    •

Key Innovation: Context-dependent strategy selection. Simple tasks get fast single-model responses. Complex tasks get multi-agent collaboration. The system learns which strategy works best for each task type.

The Results



Measurement methodology: Same 6-dimension quality rubric as above. Each method ran on the same 100 tasks. Pass threshold: 0.70.



| Method | Pass Rate* | Avg Quality | Cost/Task* | TCO (1K tasks)**** |


|--------|-----------|-------------|-----------|-----------------|

| Sequential | 73% (73/100) | 0.726 | $0.195 | $12,345 |

| Reflexion | 61% (61/100) | 0.681 | $0.148 | $17,698 |

| Plan-and-Solve | 42% (42/100) | 0.637 | $0.062 | $26,162 |

| Self-Consistency | 34% (34/100) | 0.598 | $0.125 | $29,825 |

| Baseline | 19% (19/100) | 0.531 | $0.025 | $36,475 |




*Pass Rate: % of tasks scoring ≥0.70 on quality rubric


**Avg Quality: Mean score across all 100 tasks (scale 0.00-1.00)


***Cost/Task: Measured API token usage × model pricing (OpenRouter rates as of Oct 2025)


****TCO: API cost + ($60/hour developer time × 0.75 hours/failure) for 1,000 tasks



TCO calculation example (Sequential):


    

  • API cost: 1,000 tasks × $0.195 = $195
    • 

  • Failures: 27% of 1,000 = 270 tasks
    • 

  • Fix cost: 270 × $45 (0.75 hours × $60/hour) = $12,150
    • 

  • Total: $195 + $12,150 = $12,345
    • 




Key Insight: Sequential is 8x more expensive on API calls ($0.195 vs $0.025) but 66% cheaper overall ($12,345 vs $36,475) when you count developer time fixing failures.

Performance Validation



62 detections across 5 domains (15 test scenarios, each processed by 3 agents + orchestrator = 62 individual agent detections):



What "detection rate" means:


    

  • We created 15 synthetic anomaly scenarios with ground truth (we know the anomaly exists)
    • 

  • Detection rate = % of scenarios where at least one agent flagged an anomaly
    • 

  • 100% detection rate = all 15 scenarios were correctly flagged

What "confidence" means:


    

  • Each agent outputs a confidence score (0-100%) for its detection
    • 

  • Avg confidence = mean confidence across all agent detections in that domain
    • 

  • Higher confidence = stronger statistical/pattern evidence for the anomaly
    •

Why Multi-Agent Works



Problem with single-model approaches:


    

  • Statistical methods miss context (a spike might be Black Friday, not an attack)
    • 

  • Time-series models miss sudden structural changes
    • 

  • Root cause analysis requires domain knowledge
    • 




Multi-agent solution:


    

  • Pattern Analyst: "Is this statistically anomalous?"
    • 

  • Change Detective: "Is this part of a larger trend?"
    • 

  • Root Cause Agent: "What could explain this?"
    • 

  • Orchestrator synthesizes findings and flags conflicts
    • 




Example: E-commerce conversion drop


    

  • Pattern Analyst: "90% confidence anomaly (3.2σ from baseline)"
    • 

  • Change Detective: "100% confidence drift (conversion down 15% over 2 hours)"
    • 

  • Root Cause Agent: "70% confidence root cause: Payment gateway timeout (correlated with checkout latency spike)"
    • 

  • Result: High-confidence alert with actionable diagnosis
    •

What It Is



A self-improving multi-agent coding system that learns from successful patterns:



5 Specialized Agents:


    

  1. Architect - Designs solution structure
    2. 

  2. Implementer - Writes the code
    3. 

  3. Security - Identifies vulnerabilities
    4. 

  4. Tester - Generates test cases
    5. 

  5. Vision - Suggests improvements
    6. 




Quality Enforcement:


    

  • Galileo Observe scores each output (0-100)
    • 

  • 90+ threshold required to pass
    • 

  • Failed attempts trigger iterative refinement (max 3 iterations)
    • 




Knowledge Graph (Neo4j):


    

  • Stores successful patterns (code that scored 90+ on Galileo Observe)
    • 

  • Retrieves proven documentation for similar tasks
    • 

  • Tracks which docs lead to high-quality code
    • 

  • 20% quality improvement from proven doc prioritization
    • 




What "20% quality improvement" means:


    

  • Baseline: Generate code without documentation retrieval
    • 

  • With proven docs: Retrieve docs that previously led to 90+ scores, pass to agents
    • 

  • Measurement: Galileo Observe quality scores (0-100 scale)
    • 

  • Test set: 30 coding tasks, each attempted both ways
    • 

  • Result: Baseline average score 75.2, with proven docs average score 90.4 (+20.2% improvement)
    • 

  • Statistical test: Paired t-test, p = 0.003 (statistically significant)
    • 




The Innovation: Intelligent Documentation Cache



Problem: RAG systems retrieve documentation blindly. Most docs are irrelevant or misleading.



Solution: Track documentation effectiveness.



How it works:


    

  1. Task comes in: "Implement JWT authentication"
    2. 

  2. Retrieve proven patterns from Neo4j (previous 90+ scored JWT implementations)
    3. 

  3. Retrieve documentation that led to those high scores (not just generic JWT docs)
    4. 

  4. Feed proven docs + proven patterns to agents
    5. 

  5. If output scores 90+, mark those docs as "effective" for JWT tasks
    6. 




Result: 20% quality boost by frontloading relevant, proven documentation.

7. Dendritic Hackathon: Neural Network Compression



Repository: github.com/bledden/dendritic-hackathon


Status: Active - 35-epoch test running (currently epoch 2/35)


Timeline: October 2025 - January 2026 (2-month research project)


Primary Model: OpenAI Whisper-Small (240M params, speech-to-text)


Hardware: RTX 5090 (32GB VRAM) + AMD MI300X (for future scaling)



What It Is



Research project validating Perforated AI's dendritic compression on production neural networks for edge deployment:



Goal: Compress Whisper-Small (240M → ~150M params, 38% reduction) for HIPAA-compliant medical transcription on edge devices.



What "60-90% compression" means:


    

  • 60% compression = reduce model from 240M to 96M parameters (remove 144M params)
    • 

  • 90% compression = reduce model from 240M to 24M parameters (remove 216M params)
    • 

  • Compression via dendritic pruning (remove neurons with low activation patterns)
    • 




What "minimal accuracy loss" means:


    

  • Baseline: Uncompressed Whisper Small achieves 4.9% WER on LibriSpeech test-clean (OpenAI benchmark)
    • 

  • Target: <5% WER degradation = final WER <9.9% (preferably <7.5%, which is <2.6% degradation)
    • 

  • Measurement: Word Error Rate (WER) on LibriSpeech test-clean dataset
    • 

  • Validation: Our current baseline (uncompressed) measured 19.16% WER on test subset (higher than 4.9% due to smaller training set)
    • 




Technique: Perforated AI's dendritic optimization


    

  • Prunes redundant neurons based on activation patterns during training
    • 

  • Maintains accuracy by preserving critical pathways identified via gradient analysis
    • 

  • Targets 70% compression with <3% WER degradation (from our baseline of 19.16% to <22.16%)
    • 




Why Whisper?



Advantages:


    

  • Speech-to-text is easy to evaluate (Word Error Rate = WER)
    • 

  • Whisper Small (240M params) is small enough to fit in 192GB VRAM
    • 

  • Clear baseline: OpenAI reports 4.9% WER on LibriSpeech test-clean
    • 




Alternative considered: BERT, ResNet (may test as second architecture)



Technical Approach



Baseline:

```python

Uncompressed Whisper Small


model = whisper.load_model("small")


result = model.transcribe("audio.wav")


WER: 4.9% (OpenAI baseline)

```


Dendritic Compression:

```python

import PerforatedAI as pai



Wrap model with dendritic layer


model = pai.PerforatedWrapper(whisper_model)



Train with dendritic optimization


model.add_task("speech", task_id=1)


model.train(...)



Compress 70% of neurons


model.compress(compression_rate=0.70)



Evaluate


result = model.transcribe("audio.wav")


Target WER: <7.5% (baseline + <2.6% degradation)

```
